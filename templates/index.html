Drift Score Formula
drift_score = 0.50 × embedding_signal + 0.25 × recall_signal + 0.25 × consistency_signal embedding_signal = mean(centroid_shift_text, centroid_shift_image, pairwise_delta_text, pairwise_delta_image) centroid_shift = clip(1 − cosine(new_centroid, baseline_centroid), 0, 1) recall_signal = mean(relative_drop(R@1), relative_drop(R@5)) consistency_signal = mean(relative_drop(C@1), relative_drop(C@5)) drift_detected = drift_score >= 0.30 ← retraining triggered
Signals are read from the drift_reference.json artifact logged by the most recent MLflow run. Connect to the Airflow API at :8080 to see per-task drift metrics.
User Manual
Non-technical guide to using the CLIP Product Retrieval System
⬡ What is this system?
This is an intelligent product search engine powered by a CLIP vision-language model. You can search a product catalogue using natural language, a product image, or both at once. The system automatically learns from new product data every week and only retrains its model when the data has meaningfully changed.
▶ Starting the System
1
Start core infrastructure — run in the project root:
docker compose up -d
2
Place weekly CSV — put your product file in data/incoming/:
cp /path/to/products.csv data/incoming/week1.csv
The CSV must have: h_index, title, image_links, category_2, category_3, selling_price, mrp, highlights, description
3
Wait for the pipeline — Airflow checks for new CSVs every 5 minutes. Or trigger manually from this dashboard.
5
Open this dashboard at http://localhost:8000/ui (or open ui/index.html directly).
⌕ Searching for Products
Go to Product Search in the sidebar. Three modes are available:
T
Text Search — type any natural language query like "red wireless earbuds under ₹2000" and press Search.
I
Image Search — upload a product photo and the system will find visually similar products in the catalogue.
M
Multimodal — combine text and image for the most precise results. Both signals are averaged before searching.
Results show two ranked lists: by image similarity and by text similarity. Adjust Top-K to see more or fewer results.
◈ Adding New Weekly Data
1
Copy the new CSV to data/incoming/weekN.csv (where N is the week number).
2
The Airflow scheduler picks it up within 5 minutes — or click Trigger DAG Run on the Dashboard.
3
Watch the Pipeline Viz page to see each task light up as it runs.
4
When the pipeline finishes, the search index is hot-swapped automatically — no downtime.
△ Understanding Drift
The system measures how different the new week's product data is from the previous week. If the drift score reaches 0.30, the model is retrained. Otherwise, the existing model is reused (saving compute time).
Check the Drift Watch page to see the current drift score and its three component signals.

⟲ Reproducing a Past Week
Every pipeline run is fully reproducible via a Git tag + DVC data snapshot:
# List available week snapshots git tag # Restore week2's exact state (model, data, config) ./reproduce.sh week2/run-abc12345
This checks out the git tag, runs dvc pull to restore data files, and prints the corresponding MLflow run ID.
